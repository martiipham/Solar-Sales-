# Database Reference — Solar Sales AI Admin

All persistent storage uses a single SQLite database (`solar_admin.db`).
Schema is managed in `memory/database.py` — tables are created on startup if they
don't exist, and column migrations run automatically on every start.

CRUD helpers: `insert(table, data)`, `update(table, id, data)`,
`fetch_one(query, params)`, `fetch_all(query, params)`.

---

## leads

Inbound prospects from GHL webhooks, voice calls, web forms, or manual entry.

| Column | Type | Notes |
|--------|------|-------|
| `id` | INTEGER PK | Auto-increment |
| `created_at` | TEXT | `datetime('now')` default |
| `source` | TEXT | `ghl_webhook`, `manual`, `form` |
| `name` | TEXT | Full name |
| `phone` | TEXT | E.164 format preferred (`+61...`) |
| `email` | TEXT | |
| `suburb` | TEXT | |
| `state` | TEXT | Two-letter AU code (QLD, NSW, VIC, WA, SA, NT, TAS, ACT) |
| `homeowner_status` | TEXT | `owner`, `renter`, `unknown` |
| `monthly_bill` | REAL | Average electricity bill in AUD |
| `roof_type` | TEXT | `tile`, `colorbond`, `flat`, `unknown` |
| `roof_age` | INTEGER | Years |
| `score` | REAL | Qualification score 1–10 (written by qualification_agent) |
| `qualification_score` | REAL | Alias of `score` — legacy column kept for compatibility |
| `score_reason` | TEXT | 2-sentence GPT-4o explanation of the score |
| `recommended_action` | TEXT | `call_now`, `nurture`, `disqualify` |
| `pipeline_stage` | TEXT | Current GHL pipeline stage name |
| `status` | TEXT | `new`, `qualified`, `called`, `converted` |
| `call_id` | TEXT | Linked Retell call ID (set after a voice call) |
| `contacted_at` | TEXT | Timestamp of first contact |
| `converted_at` | TEXT | Timestamp of conversion |
| `client_account` | TEXT | Client ID (for multi-tenant setups) |
| `notes` | TEXT | Free-text notes |

## company_products / company_faqs / company_objections

Knowledge base tables used by the voice agent and email drafter.

**company_products** — products/services offered (solar panels, batteries, hot water):
`client_id, product_type, name, description, price_from_aud, price_to_aud, brands`

**company_faqs** — question/answer pairs:
`client_id, question, answer, category, priority`

**company_objections** — objection handling scripts:
`client_id, objection, response, priority`

## users / auth_tokens

Multi-user authentication for the swarm board.

**users**: `email, password_hash, name, role (owner/admin/client), client_id, active`

**auth_tokens**: JWT revocation list — `token_hash, user_id, expires_at, revoked`
